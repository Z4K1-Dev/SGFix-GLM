# üì° API Routes Documentation Index

## üîó **API Overview**

SGFix Project provides RESTful APIs with caching, pagination, and real-time capabilities. All APIs follow consistent patterns for error handling and response formatting. The API system now includes comprehensive support for government service applications with multi-step forms and status tracking.

---

## üèóÔ∏è **API Architecture**

### üìä **Response Format**
```typescript
// Success Response (with pagination)
{
  data: T[],                    // Array of data
  pagination: {                 // Pagination metadata
    page: number,
    limit: number,
    total: number,
    totalPages: number,
    hasNext: boolean,
    hasPrev: boolean
  }
}

// Success Response (single item)
{
  id: string,
  // ...other fields
}

// Error Response
{
  error: string,               // Error message
  status: number               // HTTP status code
}
```

### ‚ö° **Caching Strategy**
| Endpoint | Cache Duration | Cache Key Pattern | Invalidation |
|----------|----------------|-------------------|--------------|
| `GET /api/berita` | 5 minutes | `berita:list:${params}` | On POST/PUT/DELETE |
| `GET /api/pengaduan` | 3 minutes | `pengaduan:list:${params}` | On POST/PUT/DELETE |
| `GET /api/layanan` | 3 minutes | `layanan:list:${params}` | On POST/PUT/DELETE |
| `GET /api/kategori` | No cache | - | - |
| `GET /api/notifikasi` | No cache | - | On create |

### üì° **Socket Events**
| Event | Direction | Purpose | Data |
|-------|-----------|---------|------|
| `join-admin` | Client ‚Üí Server | Join admin room | - |
| `join-user` | Client ‚Üí Server | Join user room | - |
| `join-room` | Client ‚Üí Server | Join specific room | `room: string` |

| `pengaduan-update` | Server ‚Üí Client | Report status update | `{ pengaduanId, status, timestamp }` |
| `layanan-update` | Server ‚Üí Client | Service status update | `{ layananId, status, timestamp }` |
| `berita-update` | Server ‚Üí Client | News update | `Berita` |
| `send-notification` | Client ‚Üí Server | Send notification | `{ type, message, room? }` |
| `update-pengaduan-status` | Client ‚Üí Server | Update report status | `{ pengaduanId, status, room? }` |
| `update-layanan-status` | Client ‚Üí Server | Update service status | `{ layananId, status, room? }` |
| `new-balasan` | Client ‚Üí Server | New reply to report/service | `{ type, id, balasan, room? }` |
| `heartbeat` | Client ‚Üí Server | Keep-alive signal | - |

### üîå **Socket Configuration**
```typescript
// Client connection
const socket = io('/api/socket', {
  path: '/api/socket',
  transports: ['websocket', 'polling'],
  reconnection: true,
  reconnectionAttempts: 5,
  timeout: 5000,
  pingTimeout: 60000,
  pingInterval: 2500
})

// Listen for notifications
socket.on('notification', (data) => {
  console.log('New notification:', data)
})

// Listen for report status updates

  console.log('Report status updated:', data)
})

// Listen for service status updates
socket.on('layanan-status-updated', (data) => {
  console.log('Service status updated:', data)
})
```
